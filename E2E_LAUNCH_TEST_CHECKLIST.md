# 🚀 E2E Launch Test Checklist — 7esab.com v3.62.3

**الهدف:** التأكد قبل أى عميل مدفوع أن كل المسارات الحرجة تعمل من البداية للنهاية بدون أخطاء.

**كيف تستخدمه:** نفِّذ كل سيناريو بالترتيب. عند أى خطأ أو سلوك غريب، **توقَّف وأخبرنى فوراً** — سأُصلحه قبل أن نُكمل.

**المدة المتوقَّعة:** 2-3 ساعات.

---

## 🟦 السيناريو 1 — التسجيل والإعداد الأولى (20 دقيقة)

### 1.1 Signup جديد
- [ ] افتح **نافذة متخفية (Incognito)**
- [ ] اذهب لـ `https://7esab.com` ثم اضغط "تسجيل"
- [ ] استعمل إيميل جديد لم تستعمله قبل (مثلاً `gmail+test1@gmail.com`)
- [ ] أكمل نموذج التسجيل (اسم الشركة، البريد، كلمة مرور)
- [ ] **توقَّع:** إيميل تفعيل يصلك خلال 30 ثانية
- [ ] اضغط رابط التفعيل

**❓ توقَّف وأخبرنى إذا:**
- لم يصلك إيميل التفعيل
- ظهر خطأ "tax_number column missing" أو ما يشبهه
- تم التحويل لصفحة فارغة أو 404

### 1.2 الوصول للوحة الأولى
- [ ] بعد التفعيل، يجب أن تُحوَّل لـ `/dashboard` مباشرة
- [ ] الـ Sidebar يعرض كل الأقسام (المبيعات، المشتريات، إلخ)
- [ ] الـ Sidebar يعرض اسم الشركة بشكل صحيح
- [ ] لا توجد أخطاء حمراء فى الـ Console (F12)

**❓ توقَّف وأخبرنى إذا:**
- ظهر "Service Worker timeout" أو "503"
- لم تظهر الشركة فى أعلى الصفحة
- Console يعرض أخطاء حمراء (انسخها لى)

### 1.3 إعدادات أولية
- [ ] اذهب لـ `/settings/users` → يجب أن ترى نفسك كـ owner
- [ ] افتح `/settings/branches` → يجب أن يكون هناك فرع رئيسى افتراضى
- [ ] افتح `/settings/warehouses` → مخزن افتراضى موجود
- [ ] افتح `/settings/cost-centers` → مركز تكلفة افتراضى موجود

**❓ توقَّف إذا:** أى من هذه فارغ (يعنى السايناب لم يُنشِئ افتراضيات).

## 🟩 السيناريو 2 — دورة بيع كاملة (30 دقيقة)

- [ ] احفظ → يظهر فى القائمة

### 2.3 إنشاء فاتورة مبيعات

### 2.5 تقرير الذمم
- [ ] `/reports/customers-aging` (إن وُجد) → العميل يجب أن يظهر بـ 100 ج.م متبقية

## 🟨 السيناريو 3 — النسخ الاحتياطية الكاملة (30 دقيقة)

هذا الجزء **حرج جداً** لأن أهم وعد للعميل هو "بياناتك آمنة".

### 3.1 تصدير بدون تشفير
- [ ] `/settings` → اضغط "تصدير نسخة احتياطية"
- [ ] فى الـ dialog، اضغط "تصدير بدون تشفير"
- [ ] **توقَّع:** ملف JSON يُحمَّل + Toast نجاح + صف جديد فى جدول السجل
- [ ] افتح الملف → تأكد من وجود `metadata.checksum` + 155 جدول

### 3.2 تصدير مُشفَّر
- [ ] اضغط "تصدير" مرة أخرى
- [ ] فعِّل التشفير + اكتب passphrase قوية (مثلاً `Test@12345678`)
- [ ] صدِّر
- [ ] افتح الملف فى محرر نصوص → يجب أن ترى `ciphertext` فقط، لا بيانات تجارية مرئية

### 3.3 محاولة استعادة فى شركة أخرى (cross-tenant)
- [ ] سجِّل خروج، سجِّل دخول لحساب شركة أخرى (لو عندك)
- [ ] حاول رفع ملف backup الأول
- [ ] **توقَّع:** رسالة "هذه النسخة الاحتياطية تخص شركة أخرى" + 403
- [ ] الاستعادة لا تتم

### 3.4 تحقق من سجل المراجعة
- [ ] `/settings/audit-log` → يجب أن ترى صف "تصدير نسخة احتياطية" بـ اسم المستخدم
- [ ] اضغط "تفاصيل" → ترى "بيانات إضافية" (5 حقول)

### 3.5 تحميل من السجل
- [ ] فى `/settings`، فى جدول السجل، اضغط "تحميل" على صف قديم
- [ ] **توقَّع:** الملف يُحمَّل عبر signed URL خلال 5 دقائق

**❓ توقَّف إذا:**
- التشفير لا يفك بكلمة المرور الصحيحة
- الـ cross-tenant guard لا يعمل (تم الاستعادة فعلاً!)
- زر "تحميل" لا يعمل أو يُرجِع خطأ 403

## 🟧 السيناريو 4 — الاشتراك والدفع (45 دقيقة)

### 4.1 صفحة الباقات
- [ ] `/settings/billing` → افتح
- [ ] الباقات تظهر بأسعار EGP صحيحة
- [ ] العملة EGP فقط (لا USD)
- [ ] VAT 14% مذكور

### 4.2 محاكاة الفترة التجريبية
- [ ] إذا كنت فى trial، يجب أن يظهر "متبقى X يوم"
- [ ] Banner يظهر إذا قارب الانتهاء

### 4.3 اختبار Paymob (مع 5 جنيهات حقيقية)
**⚠️ هذا اختبار بأموال حقيقية.** استعمل بطاقتك الشخصية بمبلغ صغير ثم استرده.

- [ ] اضغط "تجديد" أو "ترقية"
- [ ] أدخل بيانات البطاقة (بطاقتك الشخصية)
- [ ] أدفع 5-10 ج.م
- [ ] **توقَّع:** تحويل لصفحة Paymob → دفع → رجوع لـ /settings/billing مع رسالة نجاح
- [ ] افتح `/settings/billing` → فاتورة جديدة بحالة "paid"
- [ ] تحقق من إيميل Paymob: وصل إيصال؟
- [ ] افتح `/api/billing/invoices/[id]/pdf` → يحوى PDF صحيح

**❓ توقَّف إذا:**
- لم يتم تسجيل الدفعة فى DB
- لم يصل إيصال Paymob
- الـ webhook لم يستلم event
- PDF فارغ أو خاطئ

### 4.4 محاكاة فشل دفع (للـ test mode فقط)
- [ ] أدخل بطاقة فاشلة (`4000000000000002` لـ Stripe، مكافئها فى Paymob)
- [ ] **توقَّع:** رسالة فشل واضحة، لا تسجيل دفعة فى DB

### 4.5 محاكاة تعليق
- [ ] فى DB، اضبط `companies.subscription_status = 'suspended'` يدوياً لحسابك التجريبى
- [ ] سجِّل دخول → يجب أن تُحوَّل لـ `/suspended`
- [ ] أرجع `subscription_status = 'active'` → تعود الأمور لطبيعتها
